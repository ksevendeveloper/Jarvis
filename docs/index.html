Funcionalidades Core

A arquitetura do Jarvis prioriza latência mínima, execução determinística e controle rigoroso de privilégios.

Backend Seguro

Endpoints REST e Socket.IO protegidos com autenticação JWT e validação de token strict.

Execução Controlada

Policies de segurança robustas e allowlist de comandos no endpoint /api/execute.

Feedback Visual

Efeito visual (fx-on/fx-off) sincronizado em tempo real entre web dashboard e Orb desktop.

Comunicação Real-Time

Infraestrutura bidirecional com Socket.IO para telemetria e estados concorrentes.

Self-Hosted & Local

Controle total sobre dados e execução. Sem dependência de processamento externo obrigatoriamente.

Hardening Ativo

Projetado com mindset de segurança defensiva, isolando escopos de execução de shell.

1. Autenticação e Proxy

Requisições externas passam por uma camada de proxy reverso e validação de assinatura JWT estrita antes de tocar o roteador principal do FastAPI.

2. Validação de Policy

O comando ou pedido requisitado é checado contra uma base de políticas ativas e allowlist. Sandboxing em nível de aplicação impede code injection.

3. Estado Distribuído

Uma vez processado, o Engine gera eventos WebSockets assíncronos transmitidos em pub-sub para o Desktop Orb, Dashboard Web e displays conectados.
